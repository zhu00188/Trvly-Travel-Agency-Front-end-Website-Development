Spacing System

The spacing system is built on an 8px base unit to ensure consistency, scalability, and alignment across layouts and components. This industry-standard approach provides optimal visual rhythm and mathematical precision.

Spacing Scale (Base Unit: 8px)

8px / 16px / 24px / 32px / 40px / 48px / 72px

Section Spacing

Defines vertical rhythm between major content blocks. On mobile (<768px), reduces to 40px.

72px
Usage: Major section padding (top/bottom), hero banner spacing
40px
Usage: Sub-section spacing, content block separation

Layout Spacing

Controls grid gaps and column spacing

32px
Usage: Grid column gaps, card grid spacing
16px
Usage: Feature list gaps, form field spacing, compact grid spacing, social media icon gaps

Component Spacing

Standardizes padding and internal spacing. On mobile (<768px), component padding reduces to 15-20px.

32px
Usage: Button horizontal padding (left/right)
16px
Usage: Input field padding, small card padding, badge padding, tag containers, compact elements
8px
Usage: Small button padding, tight component spacing, icon containers, minimal padding for micro-elements

Content Spacing

Ensures readable spacing between text elements. On mobile (<768px), content margins reduce to 5-10px.

16px
Usage: Heading bottom margin, content block separation
8px
Usage: Paragraph bottom margin, list item spacing, button icon margin, inline element spacing, badge spacing, tight text separation

Accessibility

Trvly is designed to meet WCAG 2.1 Level AA standards, ensuring the platform is usable by everyone, including people with disabilities.

WCAG & AODA Compliance

Accessibility was prioritized from the start, not as an afterthought. Every design decision and code implementation was evaluated against WCAG 2.1/2.2 Level AA and AODA compliance standards. Achieved 99% Lighthouse Accessibility Score.

Accessibility Checklist

Semantic HTML

Proper use of <header>, <main>, <footer>, <nav>, <article>, and <section> elements

ARIA Labels

All interactive elements have descriptive aria-labels (e.g., "Play video", "Share on Facebook")

Keyboard Navigation

All functionality accessible via keyboard with visible focus states

Color Contrast

All text meets WCAG AA minimum contrast ratio of 4.5:1

Alt Text

All images have descriptive alt attributes for screen readers

Heading Hierarchy

Proper h1-h6 structure with no skipped levels

Form Labels

All form inputs have associated labels for screen reader compatibility
