Overview

Designing a Thoughtful Digital Experience for Modern Travelers

To help users navigate overwhelming travel options, I designed Trvly as a front-end platform that combines destination search, interactive tour filtering, and curated global travel blogs within a responsive, accessible interface, supporting informed exploration and planning.

Project Purpose

The purpose of Trvly was to support real user decision-making in the travel space. Rather than building a transactional booking system, the project focuses on the earlier, more influential stages of the user journey—discovery, comparison, and planning.

Core Challenges

The challenge was to design a travel platform that felt visually engaging without sacrificing clarity or usability. The experience needed to support discovery, comparison, and planning across different user intents while remaining responsive and accessible on all devices.

Delivered Results

Mobile‑first responsive interface built with Bootstrap 4 and custom styling; Design system with 40+ reusable UI components; Destination discovery, tour planning, and booking interface simulations; User account system with dashboard, payments, and reviews; WCAG 2.1 / 2.2 AA accessibility‑aligned design and code.

Problem & Solution

Challenges, Solutions & Deliverables

Key Challenges

  • Competitive Travel Market: The platform needed to feel aspirational and visually engaging while remaining clear, trustworthy, and easy to navigate in a landscape dominated by large booking services.
  • Diverse User Goals: Users arrive with different intentions—from casual inspiration to detailed comparison—requiring flexible yet intuitive flows that support multiple planning mindsets.
  • Mobile‑First Expectations:Most travel research happens on mobile devices, making responsive behavior, performance, and touch usability essential rather than optional.
  • Accessibility Gaps: Many travel platforms fail to meet accessibility standards, creating barriers for users relying on assistive technologies.
  • Long-Term Scalability:The system had to support future expansion in destinations, tour types, content, and user features without requiring structural redesign.

My Solutions

  • Front-End–Driven Platform Design: Designed a discovery-first travel platform centered on destinations, tour comparison, and planning rather than transactional booking, aligning the experience with real user decision-making behavior.
  • Clear Information Architecture & User Flows: Defined a clear page hierarchy and navigation system covering destinations, tour plans, blogs, and user accounts, enabling intuitive exploration.
  • Mobile-First Responsive Layout: Implemented a responsive layout using Bootstrap 4 with custom styling to ensure consistent usability, readability, and interaction quality across all device sizes.
  • Accessibility-First Implementation: Integrated WCAG 2.1 / 2.2 AA and AODA principles directly into design decisions and front-end implementation through semantic HTML, ARIA labeling, keyboard navigation, and contrast-safe color usage.
  • Component-Based UI System: Developed 40+ reusable UI components and front-end patterns, creating a flexible foundation that supports future feature growth and backend integration without reworking the core interface.

My Design & Development Process

1 - Research & Discovery

I began by examining how modern travelers research destinations, evaluate travel options, and build confidence before making a booking decision. The focus was on understanding user motivations, planning mindsets, and the types of information people seek at different stages of the travel planning process, rather than on page structure or feature definition.

5 - Front-End Development & Cross-Device Testing

Building on the wireframes, I translated the validated layouts and user flows into a fully responsive front-end using HTML5, CSS3, Bootstrap 4, JavaScript, and jQuery. Development focused on creating modular, reusable UI components that support destination discovery, tour plan filtering, and content-heavy pages such as blogs and travel guides.

Interactive features—including search, filtering by destination, departure location, price range, travel type, and promotional packages—were implemented entirely on the front end to simulate real-world travel exploration and comparison behaviors. Special attention was given to semantic markup, visual hierarchy, and performance-conscious styling to ensure clarity across dense information layouts.

The interface was tested across multiple screen sizes and devices to validate responsive behavior, accessibility, and usability, ensuring that key interactions and content remained intuitive whether users were browsing on mobile, tablet, or desktop.

Accessibility

WCAG & AODA Compliance

Accessibility was treated as a foundational part of Trvly’s design system rather than a post-launch checklist. From color selection and typography to component behavior and semantic markup, every design and front-end decision was evaluated against WCAG 2.1 / 2.2 Level AA and AODA standards to ensure inclusive access across devices and assistive technologies. As a result, key pages consistently achieve a 99% Google Lighthouse Accessibility score.

WCAG & AODA Compliant

Color Contrast

All text and interactive elements meet WCAG 2.1/2.2 AA minimum contrast requirements (4.5:1),

  • Primary Orange (#c84f00) on white (#ffffff): 4.6:1 (AA)
  • Secondary Navy (#071C55) on white (#ffffff): 16.1:1 (AAA)
  • Accessible Accent (#f25e00) on dark (#071C55): 4.9:1 (AA)
  • Body Text Primary (#666666) on white (#ffffff): 5.7:1 (AA)

Screen Reader Support

Comprehensive ARIA labels and semantic HTML for assistive technology compatibility.

  • Semantic HTML5 elements (nav, main, footer)
  • ARIA landmarks and live regions
  • Alt text for all decorative and functional images
  • Descriptive link and button labels

Responsive & Mobile

Touch-friendly targets (minimum 44x44px) and optimized mobile experience.

  • All interactive elements meet 44x44px minimum
  • Adequate spacing between clickable elements
  • Responsive text sizing (16px minimum)
  • Tested across iOS and Android devices

Content Readability

Clear typography hierarchy and readable font sizes optimized for all users.

  • Base font size: 16px (body text)
  • Line height: 1.6-1.8 for readability
  • Clear visual hierarchy with heading levels
  • Proper paragraph spacing and margins
